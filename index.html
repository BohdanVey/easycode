Binary exponentiation

The complexity of this algorithm is O(logn)

Raising a to the power of n is expressed naively as multiplication by a done n−1 times: a^n=a⋅a⋅…⋅a. However, this approach is not practical for large a or n. Recall that a^(b+c)=a^b⋅a^c and a^2b=a^b⋅a^b=(a^b)^2. Let's write n in binary system, for example: 3^11=3^8⋅3^2⋅3^1 So we only need to find the sequence of numbers in which each one is a square of the previous one: 3^1=3 3^2=(3^1)^2=3⋅3=9 3^4=(3^2)^2=9⋅9=81 3^8=(3^4)^2=81⋅81=6561 The get the final answer, we multiply three of them (skipping 34 because corresponding bit in n is zero): 3^11=6561⋅9⋅3=177147. A recursive approach expresses the same idea as a recursion: a^n=(a^(n/2))^2 for even n, a^n=(a^((n−1)/2))^2⋅a for odd n, a^1=a.
